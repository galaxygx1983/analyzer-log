# Alibaba Copaw + 飞书 + LLM 智能问答系统探索

## 基于云原生架构的企业知识库智能助手部署方案

> **部署模式说明**
>
> 本文档描述的是**纯云端部署方案**：
>
> - **Alibaba Copaw**：使用阿里云托管服务
> - **Coding Plan LLM**：调用阿里云大模型API
> - **飞书**：使用飞书开放平台云服务
>
> **本地部署替代方案**：
>
> - **Copaw**：可部署至本地服务器
> - **LLM**：可本地部署开源大模型（如 Qwen、Llama、ChatGLM 等）
> - **飞书**：可替换为其他IM（如企业微信、钉钉等）
>
> 如需本地部署方案，请参考各组件官方文档进行适配。

## 一、系统概述

### 1.1 设计原则：云原生智能问答

本方案基于 **Alibaba Copaw** 平台，结合 **飞书** 即时通讯能力和 **阿里云 Coding Plan** 大模型服务，构建企业级智能问答系统：

| 组件                      | 职责            | 选型理由                         |
| ------------------------- | --------------- | -------------------------------- |
| **Alibaba Copaw**   | AI 应用开发平台 | 阿里云原生，技能编排，低代码部署 |
| **飞书**            | 用户交互入口    | 企业级 IM，消息推送，移动端支持  |
| **Coding Plan LLM** | 大模型推理服务  | 阿里云内网访问，低延迟，高并发   |
| **技能 (Skills)**   | 专业知识封装    | 可复用、可扩展的知识单元         |
| **知识库**          | 结构化文档存储  | Markdown 格式，版本管理友好      |

## 二、技能设计与实现

### 2.1 技能一：cq24-interfaces-unified

**技能定位**：重庆24号线融合平台接口知识库

**核心功能**：

- 查询54+个ISCS子系统接口规格
- 提供承包商联络人信息
- 解答接口测试规范问题
- 支持编码规则查询

**知识库结构**：

```
skill/cq24-interfaces-unified/
├── SKILL.md                          # 技能主文档
├── references/
│   ├── cq24-interfaces-all.md        # 接口总览
│   ├── cq24-interfaces-knowledge-base.md
│   ├── electronic-patrol/            # 电子巡更系统
│   │   ├── FAQ.md
│   │   └── original-document.md
│   ├── iscs-ats-interface/           # ATS接口
│   │   ├── FAQ.md
│   │   └── original-document.md
│   ├── iscs-video-analysis/          # 智能视频分析
│   │   ├── FAQ.md
│   │   └── original-document.md
│   ├── vehicle-gateway-iscs/         # 车辆网关
│   │   ├── FAQ.md
│   │   └── original-document.md
│   └── ... (其他子系统接口)
```

**典型问答场景**：

```
用户：ISCS与ATS接口的端口号是多少？

系统：
根据接口规格书 CQ24-ISCS-01-116，ISCS与ATS接口使用以下端口：

• 实时信息端口：2100
• 非实时信息端口：2200

物理接口：冗余 100M/1000M 以太网电口，RJ45，2个
通信协议：TCP/IP，IEEE802.3
心跳周期：1秒
超时时间：3秒（连续3个周期未收到数据视为中断）

参考文档：references/iscs-ats-interface/FAQ.md
```

### 2.2 技能二：ics-protocol-analyzer

**技能定位**：工控协议通信报文解析与故障诊断

**核心功能**：

- 解析Modbus TCP/RTU报文
- 分析IEC 104/101通信帧
- 诊断S7、BACnet、MQTT协议
- 验证CRC/LRC校验
- 生成通信质量统计报告

**知识库结构**：

```
skill/ics-protocol-analyzer/
├── SKILL.md                          # 技能主文档
├── README.md                         # 使用说明
├── references/
│   ├── modbus.md                     # Modbus协议规范
│   ├── iec104.md                     # IEC 104协议规范
│   ├── iec101.md                     # IEC 101协议规范
│   ├── iec61850.md                   # IEC 61850协议规范
│   ├── s7.md                         # S7协议规范
│   ├── bacnet.md                     # BACnet协议规范
│   ├── mqtt.md                       # MQTT协议规范
│   └── diagnosis.md                  # 故障诊断指南
└── scripts/                          # 解析工具脚本
    ├── cli.py                        # 统一CLI入口
    ├── parse_hex.py                  # 十六进制解析
    ├── analyze_log.py                # 日志分析
    ├── parsers/                      # 协议解析器
    │   ├── modbus.py
    │   ├── iec104.py
    │   ├── iec101.py
    │   └── ...
    └── tests/                        # 单元测试
```

**典型问答场景**：

```
用户：解析这个Modbus报文：01 03 00 00 00 0A C4 0B

系统：
## 报文解析结果

**协议**: Modbus RTU
**方向**: 请求
**时间**: 2024-03-24 10:30:45

| 字段 | 十六进制 | 值 | 说明 |
|------|----------|-----|------|
| 从站地址 | 01 | 1 | 目标设备地址 |
| 功能码 | 03 | 3 | 读保持寄存器 (Read Holding Registers) |
| 起始地址 | 00 00 | 0 | 寄存器起始地址 |
| 读取数量 | 00 0A | 10 | 读取10个寄存器 |
| CRC校验 | C4 0B | - | ✅ 校验通过 |

**诊断**: 正常请求报文
**预期响应**: 从站应返回 1 + 2*10 + 2 = 23 字节数据

如需继续分析响应报文，请提供。
```

## 三、使用说明

### 3.1 飞书交互方式

**单聊模式**：

```
用户: @智能助手 ISCS与BAS接口是什么协议？

机器人: 
ISCS与BAS（环境与设备监控系统）接口使用 **Modbus TCP** 协议。

接口详情：
• 接口位置：云平台
• 接口类型：RJ45
• 协议：modbustcp
• 规格书状态：已签订
• 厂商：宝信

如需了解更多接口信息，请告诉我具体系统名称。
```

**群聊模式**：

```
用户: @智能助手 解析这个报文 01 03 00 64 00 01 C4 1B

机器人:
@用户 报文解析结果如下：

**协议**: Modbus RTU 请求
**从站地址**: 1
**功能码**: 03 (读保持寄存器)
**起始地址**: 100 (0x0064)
**读取数量**: 1
**CRC校验**: ✅ 通过

这是一个读取单个保持寄存器的请求报文。
```
