# 边缘侧分布式日志管理系统技术实现报告

## 1. 系统概述

本系统是一个边缘侧分布式日志管理演示系统，实现了从日志产生、存储、分析到持久化的完整链路。系统采用两道分析机制：第一道为规则分析（快速匹配已知问题），第二道为大模型分析（深度分析未知问题）。

## 3. 日志器实现

### 3.1 核心结构

**文件位置**: `pkg/logger/logger.go`

```go
type LogEntry struct {
    Name     string                 `json:"name"`     // 服务名称
    Hostname string                 `json:"hostname"` // 节点ID
    Pid      int                    `json:"pid"`      // 进程ID
    Level    int                    `json:"level"`    // Pino数字级别
    Msg      string                 `json:"msg"`      // 消息
    Time     string                 `json:"time"`     // ISO时间
    Svc      string                 `json:"svc"`      // 服务名
    Node     string                 `json:"node"`     // 节点ID
    Fields   map[string]interface{} `json:"fields,omitempty"`
}

type SimpleLogger struct {
    redis     *redis.Client  // Redis客户端
    stream    string         // Stream名称
    nodeID    string         // 节点ID
    service   string         // 服务名
    localFile *os.File       // 本地备份文件
    mu        sync.Mutex     // 写锁
}
```

### 3.2 Pino日志级别

Pino使用数字表示日志级别（与Bunyan兼容）：

| 级别 | 数值 | 说明 |
|------|------|------|
| TRACE | 10 | 跟踪信息 |
| DEBUG | 20 | 调试信息 |
| INFO | 30 | 一般信息 |
| WARN | 40 | 警告信息 |
| ERROR | 50 | 错误信息 |
| FATAL | 60 | 致命错误 |

### 3.3 双写机制

```go
func (l *SimpleLogger) Log(ctx context.Context, level, msg string, fields map[string]interface{}) {
    entry := LogEntry{
        Name:     l.service,
        Hostname: l.nodeID,
        Pid:      os.Getpid(),
        Level:    levelToBunyan(level),
        Msg:      msg,
        Time:     time.Now().UTC().Format(time.RFC3339),
        Svc:      l.service,
        Node:     l.nodeID,
        Fields:   fields,
    }

    data, _ := json.Marshal(entry)

    // 1. 异步写入Redis (失败不阻塞)
    go func() {
        l.redis.XAdd(ctx, &redis.XAddArgs{
            Stream: l.stream,
            MaxLen: 10000, // 保留最近1万条
            Values: map[string]interface{}{"data": string(data)},
        })
    }()

    // 2. 同步写入本地文件 (确保不丢)
    l.mu.Lock()
    l.localFile.Write(append(data, '\n'))
    l.mu.Unlock()
}
```

## 4. Pino应用细节

### 4.1 Pino-Pretty集成

**文件位置**: `cmd/logctl/main.go`

Pino-pretty是Pino日志的格式化工具，用于将JSON日志转换为可读格式。

#### 命令行调用

```go
func pipeToPino(ctx context.Context, redisAddr string) {
    // 使用npx运行pino-pretty（支持本地和全局安装）
    cmd := exec.CommandContext(ctx, "npx", "pino-pretty", "-t")
    cmd.Stdout = os.Stdout
    cmd.Stderr = os.Stderr

    stdin, _ := cmd.StdinPipe()
    cmd.Start()

    // 从Redis读取日志并写入pino-pretty
    for {
        result, _ := rdb.XRead(ctx, &redis.XReadArgs{
            Streams: []string{"logs:stream", lastID},
            Block:   1000,
            Count:   100,
        }).Result()

        for _, stream := range result {
            for _, msg := range stream.Messages {
                if data, ok := msg.Values["data"].(string); ok {
                    stdin.Write([]byte(data + "\n"))
                }
            }
        }
    }
}
```

```javascript
fastify.get('/api/logs', async (req, reply) => {
  const { level, service, node, limit = 100, offset = 0 } = req.query
  
  // 从 Redis Stream 读取日志
  const logs = await redis.xRange('logs:stream', '-', '+', { COUNT: limit })
  
  // 过滤处理
  let filtered = logs
  if (level) {
    filtered = filtered.filter(log => {
      const data = JSON.parse(log.message.data)
      return data.level >= parseInt(level)
    })
  }
  if (service) {
    filtered = filtered.filter(log => {
      const data = JSON.parse(log.message.data)
      return data.svc === service
    })
  }
  if (node) {
    filtered = filtered.filter(log => {
      const data = JSON.parse(log.message.data)
      return data.node === node
    })
  }
  
  return { logs: filtered, total: filtered.length }
```

```go
func NewPostgresStorage(cfg PostgresConfig) (*PostgresStorage, error) {
    // 连接到默认postgres数据库
    defaultDB, _ := sql.Open("postgres", defaultDSN)

    // 检查数据库是否存在
    var exists bool
    defaultDB.QueryRow(
        "SELECT EXISTS(SELECT 1 FROM pg_database WHERE datname = $1)",
        cfg.Database,
    ).Scan(&exists)

    // 不存在则创建
    if !exists {
        defaultDB.Exec(fmt.Sprintf("CREATE DATABASE %s", cfg.Database))
    }

    // 连接目标数据库
    db, _ := sql.Open("postgres", dsn)
    return &PostgresStorage{db: db}, nil
}
```

### 8.1 规则结构

```go
type Rule struct {
    ID          string       // 规则ID
    Name        string       // 规则名称

    Type        RuleType     // 规则类型

type Condition struct {
```

| UNKNOWN-001 | 未知错误类型 | error | high | level=error (需要LLM分析) |

### 8.4 条件匹配算法

```go
func (a *Analyzer) matchCondition(log map[string]interface{}, cond Condition) bool {
    value := a.getFieldValue(log, cond.Field)

    // 特殊处理level字段（支持数字和字符串）
    if cond.Field == "level" {
        return a.matchLevelCondition(value, cond)
    }

    switch cond.Operator {
    case "eq":
        return fmt.Sprintf("%v", value) == fmt.Sprintf("%v", cond.Value)
    case "contains":
        return strings.Contains(fmt.Sprintf("%v", value), fmt.Sprintf("%v", cond.Value))
    case "matches":
        matched, _ := regexp.MatchString(fmt.Sprintf("%v", cond.Value), fmt.Sprintf("%v", value))
        return matched
    case "gt", "lt", "gte", "lte":
        return a.compareNumbers(value, cond.Value) ...
    }
}
```

### 8.5 嵌套字段访问

```go
func (a *Analyzer) getFieldValue(log map[string]interface{}, field string) interface{} {
    parts := strings.Split(field, ".")
    if len(parts) == 1 {
        return log[field]
    }

    // 处理嵌套字段，如 fields.error_type
    current := log
    for i, part := range parts {
        if i == len(parts)-1 {
            return current[part]
        }
        if next, ok := current[part].(map[string]interface{}); ok {
            current = next
        } else {
            return nil
        }
    }
}
```

## 11. 日志文件采集器 (file-collector)

### 11.1 概述

**文件位置**: `cmd/file-collector/main.go`

file-collector 是用于采集第三方程序日志文件的组件。许多第三方应用程序将日志写入本地文件，我们无法修改其代码，因此需要一个独立的采集器来监控日志文件变化并接入系统。

### 11.2 核心功能

| 功能 | 说明 |
|------|------|
| 文件监控 | 使用fsnotify实时监控日志文件变化 |
| 增量读取 | 记录读取位置，只读取新增内容 |
| 日志轮转检测 | 自动检测文件重命名/重建，从头读取新文件 |
| 多格式解析 | 支持Log4j、JSON、Syslog等多种格式 |
| 自动格式检测 | 智能识别日志格式，无需手动配置 |

### 11.3 支持的日志格式

#### Log4j/Logback格式

```
[LEVEL] YYYY-MM-DD HH:mm:ss,SSS [thread] className - message
```

示例：
```
[INFO] 2024-03-24 15:30:45,123 [main] com.example.App - Application started
[ERROR] 2024-03-24 15:30:48,456 [http-handler] com.example.Handler - Connection timeout
```

#### JSON格式

```json
{"level":30,"time":"2024-03-24T15:30:45.123Z","msg":"Application started","svc":"app"}
```

#### Syslog格式

```
Mar 24 15:30:45 hostname process[pid]: message
```

### 11.4 解析器架构

```go
// LogParser 日志解析器接口
type LogParser interface {
    Parse(line string) (map[string]interface{}, bool)
```

### 11.5 文件监控机制

```go
type FileCollector struct {
    config    Config
    parser    LogParser
    redis     *redis.Client
    positions map[string]int64  // 文件读取位置
}

func (fc *FileCollector) Start(ctx context.Context) error {
    // 1. 创建fsnotify监控器
    watcher, _ := fsnotify.NewWatcher()

    // 2. 监控日志文件所在目录
    for _, file := range fc.config.LogFiles {
        dir := filepath.Dir(file)
        watcher.Add(dir)
        fc.positions[file] = fc.getFileSize(file)  // 从文件末尾开始
    }

    // 3. 处理文件事件
    for {
        select {
        case event := <-watcher.Events:
            if event.Op&fsnotify.Write != 0 {
                fc.readNewLines(ctx, event.Name)  // 读取新行
            }
        case <-scanTicker.C:
            fc.scanFiles(ctx)  // 定期扫描
        }
    }
}
```

### 11.6 日志轮转处理

| 场景 | 检测方式 | 处理方法 |
|------|----------|----------|
| 文件被截断 | currentSize < lastPosition | 重置position为0，从头读取 |
| 文件被重命名 | fsnotify.Rename事件 | 监控新文件创建 |
| 文件被删除重建 | 文件不存在后重新出现 | 从头读取新文件 |
