# 莱钢日志存储方案说明

## 一、方案概述

本方案实现了一套高性能、可扩展的分布式日志存储系统，通过 **本地文件 + Redis + SQLServer** 三级存储架构，满足日志的实时写入、高效传输和持久化存储需求。该方案已在莱钢项目中成功应用。

## 二、核心组件

### 2.1 SmartRailPlatformManager (DLL)

**类型**: 动态链接库
**核心类**: [SmartRailManager](SmartRailPlatformManager/SmartRailManager.hpp)

#### 主要职责

- 封装 CVLog 日志组件，提供统一的日志接口
- 支持日志写入本地文件和/或远端 Redis
- 提供日志过滤机制，避免重复日志刷屏

#### 关键接口

```cpp
// 日志记录入口
void logMessage(const std::string& logFilter, 
                SR_LOG_LEVEL logLevel, 
                const std::string& trainNo, 
                bool inRedis,              // 是否写入Redis
                const std::string& logMsg);

// 配置Redis连接
void setRedistore(const std::string& host, 
                  uint16_t port, 
                  const std::string& password, 
                  const std::string& logKey);

// 设置日志文件名
bool setLogFileName(const std::string& logFileName);

// 配置日志过滤器（防止重复日志）
void setLogFilter(const std::string& log_filter, 
                  uint8_t logNumber,    // 最大记录数
                  uint8_t timeOut);     // 超时时间(秒)
```

#### 日志写入流程

```
logMessage() 
    │
    ├─▶ 检查日志过滤器 (setLogFilter配置)
    │       │
    │       ├─ 未配置过滤器 ──▶ 直接调用 WriteLog()
    │       │
    │       └─ 已配置过滤器 ──▶ 检查是否重复
    │                               │
    │                               ├─ 重复日志 ──▶ WriteLog(inRedis=false)
    │                               └─ 非重复日志 ──▶ WriteLog(inRedis=传入值)
    │
    └─▶ WriteLog()
            │
            ├─▶ m_logger.LogMessage() ──▶ 写入本地日志文件
            │
            └─▶ if(inRedis) ──▶ writeLogToRedis() ──▶ RPUSH 到 Redis List
```

#### 日志格式

日志消息格式：

```
{时间戳} [{日志级别}] [{线程ID}] {日志内容} [{日志类型}] [{车次号}]
```

示例：

```
2025-01-15 14:47:41.623 [INFO] [e1c4] DEMO_SQLServer正常退出 [DEMO_SQLServer] [134]
```

### 2.2 SmartRail_TransferLogs (可执行程序)

**类型**: 可执行程序
**部署要求**: 需与 SQLServer 运行在同一服务器上
**核心类**: [DealThread](SmartRail_TransferLogs/DealThread.cpp)

#### 主要职责

- 从 Redis List 中批量读取日志
- 解析日志格式
- 使用 BULK INSERT 高性能批量写入 SQLServer

#### 配置文件

配置文件：`SmartRail_TransferLogs.json`

```json
{
  "RedisConfig": {
    "redis_connection": [
      {
        "id": "Redis1",
        "ip": "127.0.0.1",
        "port": 6377,
        "password": ""
      }
    ]
  },
  "DBConfig": {
    "database_connection": [
      {
        "id": "SQLServer1",
        "ip": "127.0.0.1",
        "database_name": "mydb",
        "username": "gexin",
        "password": "baosight"
      }
    ],
    "application": {
      "SmartRail_TransferLogs": {
        "sql_statements": [
          {
            "id": "HitLogBulkInsertSql",
            "database": "SQLServer1",
            "sql": "BULK INSERT t_hit_log"
          }
        ]
      }
    }
  },
  "keys": [
    {
      "redis_connection": "Redis1",
      "redis_key": "HitLog",
      "sql_statement": "HitLogBulkInsertSql",
      "sql_fields": "fd_date_time,fd_message,fd_level,fd_type,fd_train"
    }
  ]
}
```

#### 处理流程

```
主循环 (svc)
    │
    ├─▶ 检查节点状态 (SMARTRAIL_RM_STATUS_ACTIVE)
    │
    └─▶ 遍历 keys 配置
            │
            └─▶ processKeyConfig()
                    │
                    ├─ 1. connectRedis() ──▶ 连接Redis
                    │
                    ├─ 2. LLEN 获取日志队列长度
                    │
                    ├─ 3. LRANGE 批量读取日志 (最大10000条)
                    │
                    ├─ 4. LTRIM 删除已读取的日志
                    │
                    ├─ 5. 正则解析日志格式
                    │
                    └─ 6. executeSQLStatements()
                            │
                            ├─ 生成临时文件 (@分隔符)
                            ├─ BULK INSERT 批量写入
                            └─ 删除临时文件
```

#### BULK INSERT 格式文件

`SmartRail_TransferLogsFormat.xml` 定义了数据导入格式，配合临时文件实现高性能批量写入。

## 三、日志存储模式

### 3.1 当前支持的模式

| 模式  | 本地文件 | Redis | SQLServer | 适用场景                     |
| ----- | -------- | ----- | --------- | ---------------------------- |
| 模式1 | ✓       | ✗    | ✗        | 普通应用日志                 |
| 模式2 | ✓       | ✓    | ✓        | 需要持久化到数据库的业务日志 |

**调用示例**：

```cpp
// 模式1: 仅写入本地文件
SmartRailPM_logMessage("filter", SR_LOG_LEVEL_INFO, "", false, "普通日志");

// 模式2: 写入本地文件 + Redis (最终入库SQLServer)
SmartRailPM_logMessage("filter", SR_LOG_LEVEL_INFO, "车次123", true, "业务日志");
```

### 3.2 待扩展模式 (建议改造)

| 模式  | 本地文件 | Redis | SQLServer | 适用场景               |
| ----- | -------- | ----- | --------- | ---------------------- |
| 模式3 | ✗       | ✓    | ✓        | 高频日志，无需本地留存 |
| 模式4 | ✓(异步) | ✓    | ✓        | 高性能要求场景         |

## 四、改造建议

### 4.1 接口通用化改造

**背景**: 当前动态库针对莱钢项目存在定制化接口，若推广至其他项目，需进行接口通用化改造。

**改造方案**:

**方案A：通用接口设计**

设计一套通用的日志接口，通过配置参数适配不同项目需求：

```cpp
// 通用日志配置结构
struct LogConfig {
    std::string module_name;           // 模块名称
    std::string log_file_path;         // 日志文件路径
  
    // Redis配置
    bool enable_redis;                 // 是否启用Redis
    std::string redis_host;
    uint16_t redis_port;
    std::string redis_password;
    std::string redis_key;
  
    // 写入模式
    enum class WriteMode {
        LOCAL_ONLY,                    // 仅本地文件
        LOCAL_AND_REDIS,               // 本地+Redis
        REDIS_ONLY,                    // 仅Redis
        LOCAL_ASYNC_AND_REDIS          // 本地异步+Redis
    } write_mode;
  
    // 内存缓冲配置
    bool enable_memory_buffer;         // 是否启用内存环形缓冲
    size_t buffer_size;                // 缓冲区大小
};

// 通用初始化接口
void initialize(const LogConfig& config);
```

**方案B：项目定制化接口**

保留莱钢定制接口，为不同项目提供独立的接口封装层：

```cpp
// 莱钢项目专用接口
namespace Laigang {
    void logMessage(SR_LOG_LEVEL level, const std::string& msg, bool inRedis);
}

// 通用项目接口
namespace General {
    void logMessage(const LogConfig& config, SR_LOG_LEVEL level, const std::string& msg);
}
```

**建议**: 优先采用方案A，通过配置化实现通用性，减少代码维护成本。

### 4.2 内存环形缓冲区

**问题**: 当前方案未设置内存环形缓冲区，若 Redis 不可用，日志会丢失。

**影响分析**:

| 场景               | 当前行为         | 风险等级 |
| ------------------ | ---------------- | -------- |
| Redis 正常         | 日志正常写入队列 | 无风险   |
| Redis 短暂不可用   | 日志丢失         | 中风险   |
| Redis 长时间不可用 | 大量日志丢失     | 高风险   |

**改造方案**:

```
┌─────────────────────────────────────────────────────────────────┐
│                      日志写入流程 (改造后)                        │
│                                                                  │
│  logMessage()                                                   │
│      │                                                           │
│      ▼                                                           │
│  ┌─────────────────────────────────────────┐                    │
│  │         内存环形缓冲区 (Ring Buffer)       │                    ││  │              容量: 可配置 (如10000条)      │                    │
│  └────────────────────┬────────────────────┘                    │
│                       │                                          │
│                       ▼                                          │
│            ┌─────────────────────┐                              │
│            │   Redis 可用检查     │                              │
│            └──────────┬──────────┘                              │
│                       │                                          │
│         ┌─────────────┴─────────────┐                            │
│         ▼                           ▼                            │
│    Redis 可用                   Redis 不可用                     │
│         │                           │                            │
│         ▼                           ▼                            │
│  ┌──────────────┐          ┌──────────────────┐                  │
│  │ RPUSH Redis  │          │ 日志暂存内存缓冲  │                  │
│  └──────────────┘          │ 等待Redis恢复后重试│                  │
│                            └──────────────────┘                  │
└─────────────────────────────────────────────────────────────────┘
```

**实现要点**:

```cpp
class LogRingBuffer {
public:
    // 写入日志 (线程安全)
    bool push(const LogEntry& entry);
  
    // 读取日志 (批量)
    size_t pop_batch(std::vector<LogEntry>& entries, size_t max_count);
  
    // 获取当前缓冲区使用量
    size_t size() const;
  
    // 检查是否已满
    bool is_full() const;
  
private:
    std::array<LogEntry, MAX_BUFFER_SIZE> m_buffer;
    std::atomic<size_t> m_head{0};
    std::atomic<size_t> m_tail{0};
    std::mutex m_mutex;
};
```

**重试机制**:

1. Redis 恢复后，自动从缓冲区读取积压日志并写入
2. 缓冲区满时，可配置策略：
   - 丢弃最旧日志 (推荐)
   - 阻塞等待 (可能影响性能)
   - 降级为仅本地文件

### 4.3 CVLog 异步写入

**目的**: 提升日志写入性能，避免阻塞业务线程

**实现思路**:

1. 引入生产者-消费者模型
2. 使用线程安全的队列缓冲日志消息
3. 后台线程批量写入文件

**代码位置**: `SmartRailManager::WriteLog()` 中的 `m_logger.LogMessage()` 调用

### 4.4 仅写入 Redis 模式

**目的**: 对于高频日志，跳过本地文件直接写入 Redis

**接口扩展建议**:

```cpp
// 扩展日志写入模式枚举
enum class LogWriteMode {
    LOCAL_ONLY = 0,       // 仅本地文件
    LOCAL_AND_REDIS = 1,  // 本地文件 + Redis
    REDIS_ONLY = 2,       // 仅Redis
    REDIS_ASYNC = 3       // Redis异步写入
};

// 扩展接口
void logMessage(const std::string& logFilter, 
                SR_LOG_LEVEL logLevel, 
                const std::string& trainNo, 
                LogWriteMode mode,        // 新增：写入模式
                const std::string& logMsg);
```

### 4.5 Redis 写入优化

**当前实现**: 同步 RPUSH

**优化建议**:

1. 使用 Redis Pipeline 批量写入
2. 引入本地队列，批量提交减少网络开销
3. 连接池管理，避免频繁创建连接

### 5.2 注意事项

1. **SmartRail_TransferLogs 必须与 SQLServer 同机部署**

   - BULK INSERT 要求格式文件和临时文件在数据库服务器本地
2. **Redis 作为日志缓冲**

   - 提供削峰填谷能力
   - 支持多应用并发写入
3. **高可用建议**

   - Redis 配置主从或集群模式
   - SQLServer 配置 AlwaysOn 或镜像
