## 🔌 通用接口模式

### 智慧运维系统通用接口架构

大多数智慧运维系统（ALM、PMS、ATS、电扶梯、车辆）采用相同的 5 类接口模式：

| 接口编码 | 接口名称 | 源系统 | 目标系统 | 协议 |
|---------|---------|--------|---------|------|
| XXX-F01 | 预警告警信息接口 | 智慧运维系统 | 融合平台 | ModBus/TCP/IP/HTTP/Kafka |
| XXX-F02 | 关键设备状态数据接口 | 智慧运维系统 | 融合平台 | ModBus/TCP/IP/HTTP/Kafka |
| XXX-F03 | 维修建议、物资建议字典接口 | 智慧运维系统 | 融合平台 | HTTP/WEB 服务 |
| XXX-F04 | 设备指标数据接口 | 智慧运维系统 | 融合平台 | HTTP/WEB 服务 |
| XXX-F05 | 设备、子系统健康度数据接口 | 智慧运维系统 | 融合平台 | HTTP/WEB 服务 |

### 健康度评分规则（通用）

**设备健康度**：满分 100 分
- 主要故障扣 4 分
- 次要故障扣 2 分
- 提示报警不扣分

**子系统健康度**：
```
健康度 = 100 - (关键设备数量/总数量) × 关键设备的 (主要报警次数×4 + 次要报警次数×2)
              - (非关键设备数量/总数量) × 非关键设备的 (主要报警次数×2 + 次要报警次数×0.5)
```

### 13. 车辆网关 (vehicle-gateway-iscs)

**文档编号**: CQL24-3326  
**版本**: V1.0  
**项目编号**: GC0128

**物理接口**:
- 位置：控制中心 (OCC) 和备用控制中心 (BOCC)
- 类型：冗余 100M 电口 × 2
- 连接方式：A-A、B-B 连接
- 通信端口：2200
- 协议：UDP/IP

**通信协议**:
- 数据链路层：IEEE 802.3 (CSMA/CD)
- 网络层：IPv4
- 传输层：UDP/IP
- 数据编码：大端序
- CRC 校验：G(x)=x^16+x^12+x^5+1，初始值 0xFFFF

**数据包结构** (车辆网关 → ISCS):

| 包类型 | 包识别号 | 车辆状态数据长度 | 总长度 |
|-------|---------|---------------|-------|
| A 包 | 0xA | 1328 字节 | 1472 字节 |
| B 包 | 0xB | 1360 字节 | 1472 字节 |
| C 包 | 0xC | 1340 字节 | 1472 字节 |
| D 包 | 0xD | 1240 字节 | 1472 字节 |
| E 包 | 0xE | 1300 字节 | 1472 字节 |

**分包数据** (COMID):
- COMID 10000: CCU→TRDP 数据 (分包 A)
- COMID 10108: CCU→HMI_0 数据 (分包 A)
- COMID 10208: TCU2→TCMS ~ TCU12→TCMS (分包 A、B)
- COMID 10308-10408: SIV、BCU、PA、HVAC、FAS、TIDS (分包 C)
- COMID 10508-10808: EDCU、LCU、BMS、PMS、ATC、TDS (分包 D、E)

**故障处理**:
- ISCS 连续 5 秒未收到数据 → 通信故障
- 车辆网关连续 5 秒未收到心跳 → 通信故障
- 心跳周期：1 秒
- 车辆状态转发延迟：≤50ms

**ISCS → 车辆网关心跳**:
```
命令类型：1 字节 (0x0C)
```

## 📝 设备台账标准化通用规则

### 时间要求
- 大多数系统：接口规格书签订后 **60 日内** 完成 RMS 导入
- 信号智能运维 (ATS)：**90 日内** 完成 RMS 导入

### 基础信息表
- 设备台账表
- 地点表
- 安装位置表
- 设备类型表

### 状态字段含义
| 状态 | 含义 |
|-----|------|
| 原有 | 从系统内导出已存在的数据（导入后不做任何操作） |
| 新增 | 往系统内添加数据 |
| 修改 | 将系统内导出的数据做基础信息更新 |
| 删除 | 将系统内导出的数据进行删除 |

### 填写规则
1. 标题红色填充的字段为必填项
2. "设备编号""ID"列无需填写，由系统自动生成
3. "出厂编号"列填写智慧运维系统中的设备唯一码
4. 设备名称、子设备名称、部件名称、子部件名称为父子结构，同一行只能存在一个名称

## 📐 编码规范

### 分析区域与分组编码 (20 位)

| 位码 | 含义 | 长度 | 说明 |
|-----|------|------|------|
| 1-6 | 行政区划代码 | 6 | 国标 GB/T 2260，例：重庆市南岸区为 500108 |
| 7-8 | 项目类型 | 2 | 例：11=地铁交通，12=机场，13=智慧园区 |
| 9-10 | 线路/项目编号 | 2 | 例：地铁 24 号线 |
| 11-13 | 站点/位置编号 | 3 | 线路内的具体位置 |
| 14-15 | 主功能区 | 2 | 例：00=全站，01=站台，02=站厅 |
| 16-18 | 子区域/分组序号 | 3 | 主功能区内自定义区域的流水号 |
| 19-20 | 区域类型 | 2 | 例：01=物理区域，02=逻辑分组 |

### 摄像头编码
- **cameraNationalCode**: 20 位国标编码 (GB/T 28181)
- **hikPlanID**: 5 位海康平台规划 ID（全线唯一）

### 字符编码要求
- 所有非 ASCII 字符串使用 **UTF-8** 编码
- 确保跨平台兼容性（Windows、Linux）
