# 设备信息标准化

## 设备信息标准化工作要求

为满足重庆市轨道集团设备管理的标准化要求，通信智能运维承包商应满足轨道设备检修管理系统（RMS）的设备台账录入标准，并在接口规格书签订后60日内，提供信号智慧运维系统所管理设备的标准化设备台账，并在轨道设备检修管理系统（RMS）完成导入工作。

本文档中所涉及"专业"、"系统"、"设备类别"、"设备编码"均以轨道设备检修管理系统（RMS）内的标准化设备信息为主数据标准。

## 设备信息标准化规则说明

1\. 设备台账基础信息说明

1）设备台账表：设备信息标准化台账的信息收集模板，参考《附件一：设备台账基础信息》"设备台账"标签页签，需按格式填写设备台账数据，导入系统后，生成标准化设备台账，并可导出RMS系统设备编码与智慧运维系统设备唯一码的关联关系；

2）地点表：作为设备台账表中"线路"、"地点类型"、"地点"字段的数据字典，参考《附件一：设备台账基础信息》"地点"标签页签提供的地点信息；

3）安装位置表：作为设备台账表中"安装位置"字段的数据字典，参考《附件一：设备台账基础信息》"安装位置"标签页签；

4）设备类型表：作为设备台账表中"专业"、"系统名称"、"设备类型"，"子设备类型"字段的数据字典，参考《附件一：设备台账基础信息》"设备类型"标签页签；

2、设备台账填写字段规则说明

1）《附件一：设备台账基础信息》"设备台账"标签页签中标题红色填充的字段为必填项，其他非必填项可根据台账信息尽可能完善；"设备编号""ID"列无需填写，导入系统后由系统自动生成；"出厂编号"列填写智慧运维系统中的设备唯一码。

2）状态：

原有：指从系统内导出已存在的数据（执行导入后不会对标注"原有"状态的数据做任何操作）；

新增：指往系统内添加数据（执行导入后标注"新增"状态的数据会被新增至系统）

修改：指将系统内导出的数据做基础信息表更（执行导入后会对标注"修改"状态的数据基础信息根据表格内容进行更新）

删除：指将系统内导出的数据进行删除（执行导入后会对标注"删除"状态的数据从系统内删除）

3）专业名称、系统名称、设备类型、子设备类型：参照《附件一：设备台账基础信息》"专业、设备类型"进行对号入座；

4）设备名称、子设备名称、部件名称、子部件名称：为父子结构，同一行只能存在一个名称；填写设备名称后，系统名称、设备类型2个字段均为必填；填写子设备名称后，系统名称、设备类型、子设备类型3个字段均为必填；填写部件、子部件名称后，仅需填写系统名称。

#### 设备、子系统健康度数据接口(ISCS-ALM-F05)

> 接口功能具体如下：

5.1-1 接口功能表

+:----:+:--------------------------------:+:--------------------------------------------------------:+
| 序号 | 功能需求描述                                                                                |

|      +----------------------------------+----------------------------------------------------------+
|      | 功能需求                         | ISCS调用接口，获取智慧运维系统相关设备、子系统健康度数据 |
|      +----------------------------------+----------------------------------------------------------+
|      | ISCS实现功能                     | 调用接口，并更新相关设备、子系统健康度数据               |
|      +----------------------------------+----------------------------------------------------------+
|      | 智慧运维系统实现功能             | 基于提供的统计设备、子系统健康度，并提供相关数据。       |
+------+----------------------------------+----------------------------------------------------------+

设备及子系统健康度评分标准如下，最终健康度评价结果以智慧运维系统为准。

> 1、设备和系统健康度评分标准
>
> 1）设备健康度评分
>
> 健康度评分规则：设备健康度评分满分100分，设备主要故障扣4分，次要故障扣2分，提示报警不扣分。（主要报警影响设备运行，次要报警影响设备部分功能，提示报警不影响设备运行和功能，起提示作用）
>
> 健康度计算方式:根据设备预警告警记录，按照健康度评分规则进行计算。
>
> 2）子系统健康度评分
>
> 健康度评分规则：系统健康度评分按月度、站点进行评价，根据当月各系统故障报警次数和设备数量占比、故障扣分标准进行计算。下个月系统恢复满分，按照以上方式重新进行健康度评价。年度系统的健康度可以从健康度变化趋势查看。

健康度计算方式:100-(关键设备数量/总数量)\*关键设备的（主要报警次数\*4+次要报警次数\*2）-(非关键设备数量/总数量)\*非关键设备的（主要报警次数\*2+次要报警次数\*0.5）

- 输入：

  ----------------------------------------------------------------------------
  **序号**   **字段名称**         **字段说明**                 **类型**
  ---------- -------------------- ---------------------------- ---------------
  1          statisticStartTime   统计开始时间                 String

  2          statisticEndTime     统计结束时间                 String

  3          majorCode            专业代码                     String

  4          majorName            专业名称                     String
  ----------------------------------------------------------------------------

- 输出：

  -------------------------------------------------------------------------
  **序号**   **字段名称**     **字段说明**                  **类型**
  ---------- ---------------- ----------------------------- ---------------

  2          Info             接口状态说明                  String

  3          systemCode       设备分类/子系统代码           String

  4          systemName       设备分类/子系统名称           String
