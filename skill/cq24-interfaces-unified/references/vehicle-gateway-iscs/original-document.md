### 3.1.2 动态交互描述

车辆网关计算机A机、B机与ISCS接口设备A机、B机采用A-A、B-B的连接方式。信息交互方式如下：

- 车辆网关不区分主备，同时向ISCS接口设备发送车辆TCMS信息，综合监控内部保证只采用车辆网关一系的状态
- 综合监控A、B机向车辆网关A、B发送心跳信息
- 双方通信端口号为2200

**发送周期：**
- ISCS接口机主备机周期1s向车辆网关主备机发送心跳信息
- 车辆网关主机收到TCMS信息后，在50ms内将其转发给ISCS接口机主备机

### 3.1.3 通信故障处理

- 如果ISCS系统连续5秒未收到车辆网关的数据，则认为通信故障；但心跳信息需保持发送
- 如果车辆网关连续5秒未收到ISCS的心跳信息，则认为通信故障，不再发送车辆状态信息

### 3.1.4 电气接口

依照IEEE802.3以太网标准执行。

### 3.1.5 IP地址分配

车辆网关与综合监控系统互连的主机的IP地址由ATS统一分配。

## 3.2 通信层次描述

### 3.2.1 数据链路层

数据链路层包括LLC（逻辑链路控制）子层和MAC（媒体访问控制）子层。MAC层基于IEEE 802.3标准，采用CSMA/CD访问方式，每个设备都使用1个MAC地址。

MAC头由14个字节组成，1个帧校验序列(4字节)将被加在Ethernet帧后面。

### 3.2.2 网络层

使用IPv4协议作为网络层的协议。

### 3.2.3 传输层

使用UDP/IP协议作为传输协议。

### 3.2.6 CRC校验

说明：经校验方程计算后得到的一个两字节无符号整形数，该整形数存放时交换高低字节，即先存低字节后存高字节。

校验计算包括该帧从"源设备类型"到"应用数据"的所有字节。

校验方程为：G(x)=x^16+x^12+x^5+1 初始值为：0xFFFF

# 4 应用层数据

## 4.1 车辆网关→ISCS

车辆网关给ISCS发送车辆状态数据、报警数据。数据编码采用大端序，即高有效位存于低位，首先发送。

协议格式：

| 字段 | 长度 | 说明 |
| --- | --- | --- |
| 源设备类型 | 1字节 | 固定值 |
| 源设备编号 | 1字节 | 设备编号 |
| 目的设备类型 | 1字节 | 固定值 |
| 目的设备编号 | 1字节 | 设备编号 |
| 应用数据长度 | 2字节 | 应用数据长度 |
| 应用数据 | N字节 | 具体数据 |
| CRC校验 | 2字节 | CRC校验值 |

## 4.2 ISCS→车辆网关
