# 概述

融合平台（ISCS）与安全作业生产管理系统（SPS）在控制中心有接口。本接口文件在双方招标文件有关融合平台（ISCS）与安全作业生产管理系统（SPS）的接口要求基础上进行细化，以指导双方系统深化设计、安装及调试工作，明确双方责任范围。

编制依据：《重庆市轨道交通24号线一期站后工程综合监控系统设备采购及集成服务项目合同》、重庆市轨道交通24号线一期站后工程融合平台系统用户需求书。

本接口规格书中涉及到的电线、电缆、光纤、熔接盒、光纤终端盒、尾纤等相关辅材及施工工作由施工单位提供与实施。

#### 施工计划预申请信息推送接口(ISCS-SPS-F01)

> 接口功能具体如下：

4.1-1 接口功能表

+------+-------------------------------------------------------------------------------------------------------------------------------------------------------+
| 序号 | 功能需求描述                                                                                                                                          |
+:====:+:================================:+:==================================================================================================================:+
| 1    | 接口编号                         | ISCS-SPS-F01                                                                                                       |
|      +----------------------------------+--------------------------------------------------------------------------------------------------------------------+
|      | 功能需求                         | ISCS将需要进行施工请点的工单计划推送至安全作业生产管理系统，由安全作业生产管理系统进行补充填报，完成施工计划的填写 |
|      +----------------------------------+--------------------------------------------------------------------------------------------------------------------+
|      | ISCS实现功能                     | 推送施工计划预申请信息                                                                                             |
|      +----------------------------------+--------------------------------------------------------------------------------------------------------------------+
|      | SPS实现功能                      | 接收施工计划预申请信息                                                                                             |
+------+----------------------------------+--------------------------------------------------------------------------------------------------------------------+

- 输入：

  -----------------------------------------------------------------------------------
  **序号**   **字段名称**            **字段说明**                     **类型**
  ---------- ----------------------- -------------------------------- ---------------
  1          repareWorkOrderNum      工单号(必填)                     String

  2          lineCode                线路编码 (必填)                  String

  3          deptCode                作业部门编码(必填)               String

  4          taskContent             作业内容(必填)                   String

  5          taskQuality             作业类别(必填)                   String

  6          buildingCode            车站编码                         String

  7          taskDate                作业日期(yyyy-MM-dd)             String

  8          planDateType            提报周期                         String

  9          currentLoginName        申报人(当前系统登录人)           String

  10         currentLoginNamePhone   申报人电话(当前系统登录人电话)   String

  11         orderType               工单类型（检修；故障）           String
  -----------------------------------------------------------------------------------

- 输出：

  -----------------------------------------------------------------------
  **序号**   **字段名称**   **字段说明**                  **类型**
  ---------- -------------- ----------------------------- ---------------
  1          状态           1成功/-1失败                  int

  2          返回信息       字符串                        String
  -----------------------------------------------------------------------
